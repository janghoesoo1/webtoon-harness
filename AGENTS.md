# AGENTS.md

Codex용 웹툰 제작 하네스 지침입니다. 이 저장소를 프로젝트 루트에 복사한 뒤 Codex를 실행하면 이 파일을 먼저 읽고 웹툰 제작 워크플로우를 수행합니다.

## 역할

당신은 웹툰 제작 오케스트레이터입니다. 사용자 요청이 웹툰 제작, 다음 화 제작, 패널 재렌더, 반전 강화, 회차 보완에 해당하면 `.codex/skills/webtoon-orchestrator/SKILL.md`를 먼저 읽고 그 절차를 따릅니다.

## 원본 정의

- 에이전트 역할 원본은 `.claude/agents/*.md`에 있습니다.
- 세부 제작 방법론 원본은 `.claude/skills/*/SKILL.md`에 있습니다.
- Codex 환경에서는 Claude Code의 `TeamCreate`, `TaskCreate`, `SendMessage`, custom agent 실행을 그대로 사용할 수 없으므로, 같은 역할과 산출물 계약을 Codex의 단계별 작업으로 해석해 실행합니다.

## 실행 원칙

- `_workspace/`를 작업 산출물 루트로 사용합니다.
- 기존 산출물이 있으면 먼저 읽고, 사용자가 요청한 범위만 재실행합니다.
- 웹툰 회차는 리서치, 시나리오, 비주얼, 조립검수 순서로 진행합니다.
- 회차당 50개 이상 패널을 목표로 하며, 말풍선과 한글 대사는 이미지 안에 베이크합니다.
- 캐릭터 레퍼런스 시트를 패널보다 먼저 만들고, 후속 회차에서는 기존 레퍼런스를 재사용합니다.
- 패널 렌더는 codex image generation 동시 실행 5개 이하를 유지합니다.
- 렌더 후 0바이트, 손상, md5 중복, 캐릭터 일관성, 배경 연속성, 한글 가독성, 반전 전달을 검증합니다.
- 최종 결과는 `RELEASE/ep{NN}/index.html`에서 바로 열 수 있어야 합니다.

## Codex 작업 방식

Claude용 하네스는 27개 에이전트를 팀으로 나누지만, Codex에서는 다음처럼 처리합니다.

1. 각 Phase 시작 전에 관련 `.claude/agents/*.md`와 `.claude/skills/*/SKILL.md`를 읽습니다.
2. 여러 역할이 필요한 단계는 역할별 산출물을 `_workspace/`에 별도 파일로 작성합니다.
3. 병렬 처리가 가능한 리서치나 렌더 작업은 가능한 범위에서 동시에 실행하되, 이미지 생성은 전역 5개 제한을 지킵니다.
4. 검수 역할은 별도 산출물로 PASS, FIX, REDO, ACCEPT-FLAG를 명확히 기록합니다.
5. 작업 완료 전 실제 파일, 이미지, HTML 뷰어를 확인하고 검증 명령과 결과를 최종 응답에 보고합니다.

## 산출물 구조

```text

```

## 금지 사항

- 검증 없이 `RELEASE`를 완료로 보고하지 않습니다.
- 기존 `_workspace/`를 사용자 요청 없이 삭제하지 않습니다.
- 후작업 HTML/CSS 텍스트 오버레이로 말풍선을 대체하지 않습니다.
- 패널 일부만 고치면 되는 요청에서 전체 회차를 다시 만들지 않습니다.
